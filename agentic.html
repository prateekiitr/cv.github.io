The Agent Wars —
OpenClaw, NemoClaw,

From a single LLM answering questions to always-on agents that act, remember, and evolve — the past six months have fundamentally redrawn what "AI" means in practice. Here's the full picture.

Context

From Chat Boxes to Autonomous Workers

For most of AI's recent history, the interaction model was simple: you type, the model responds. Impressive, but fundamentally passive. The model had no memory of yesterday, no ability to click a button, and no way to act on your behalf while you slept.

That model is being replaced — fast. What has emerged in 2025–2026 is an entirely new software layer: the agent harness. And the ecosystems being built around it — OpenClaw, NemoClaw, Hermes — are more consequential than any single model release.

💡

The key insight: Progress in practical AI is no longer only about better models. It's about better systems — how models are wrapped, given tools, fed context, and kept running autonomously over time.

Foundation

The Six Pillars of a Coding Agent Harness

Sebastian Raschka's recent deep-dive into Claude Code and Codex CLI crystallised something important: what makes coding agents feel magical is not the model. It's the harness architecture. Here are the six components that separate a coding agent from a chat window.

01

Live Repo Context

Before any work starts, the agent builds a workspace snapshot — git branch, recent commits, AGENTS.md, directory layout. Never starting cold.

02

Prompt Cache Reuse

Stable content (instructions, tool descriptions, workspace summary) is cached. Only the session-variable parts get rebuilt each turn. Cost efficiency at scale.

03

Structured Tool Use

Pre-defined named tools with typed inputs. Every call goes through validation — known tool? valid args? needs approval? path inside workspace?

04

Context Bloat Control

Clipping (no single output eats the budget) + transcript summarisation (recent = rich, old = compressed) + deduplication of repeated file reads.

05

Structured Session Memory

Two distinct layers: a full transcript (durable, resumable JSON) and working memory (distilled, task-focused). Different jobs, different update rates.

06

Bounded Subagents

The hard problem isn't spawning a subagent — it's bounding one. Subagents inherit enough context to be useful but run with constrained permissions.

🔑

Raschka's key claim: If you dropped a capable open-weight model into a Claude Code-style harness, it could likely match GPT-5 in Codex. The harness is often the differentiator — not the model weights.

The Platforms

OpenClaw, NemoClaw & Hermes —
What Each One Actually Is

Three distinct platforms now define the agentic landscape. They share DNA but serve fundamentally different purposes. Here's each one, plainly.

🐾
OpenClaw
by Peter Steinberger · Acquired by OpenAI

The foundational layer of the current agent ecosystem. OpenClaw is best described as the operating system for personal AI — Jensen Huang's exact framing at GTC 2026. It went from zero to the fastest-growing open-source project in history before its creator was hired by OpenAI.

  • Always-on local agent that runs autonomously on your machine
  • Persistent file system, tool access, memory across sessions
  • Open agent protocol — any model, any tool backend
  • AGENTS.md, SOUL.md, TOOLS.md as workspace instruction files
  • JSONL session transcripts with compaction and resumption
  • Spawns helper sessions and subagents natively

The catch: A security incident — an unsecured database allowing agent impersonation — led Meta and several large enterprises to ban it from corporate machines. This created the opening for what came next.

Foundation Layer
🔐
NemoClaw
by NVIDIA · Launched GTC 2026 · Open Source

NemoClaw is not a replacement for OpenClaw. It is a security and privacy layer that installs on top of it in a single command. Think of OpenClaw as the worker and NemoClaw as the security badge, corporate policy handbook, and compliance officer combined.

  • OpenShell runtime — every agent runs in an isolated sandbox
  • Privacy router — cloud model calls go through a proxy; your data doesn't leak
  • Policy-based guardrails — define what each agent can access, execute, report
  • Hardware-agnostic — does not require NVIDIA GPUs
  • Supports local models (Nemotron) and cloud models in hybrid mode
  • Pitched to Salesforce, Cisco, Google, Adobe, CrowdStrike pre-launch

Strategic play: NVIDIA is following Meta's Llama playbook — give the software layer away freely, build the ecosystem, trust that accelerating enterprise AI workloads will drive GPU demand anyway.

Enterprise Security Layer
⚕️
Hermes Agent
by NousResearch · Released Feb 2026 · MIT License

Where OpenClaw is a platform and NemoClaw is a security wrapper, Hermes is a personal agent that evolves with you. Built by NousResearch — the lab that pioneered tool-calling fine-tunes — it brings a different philosophy: the agent should grow more capable the longer it runs.

  • Multi-level persistent memory — learns projects, preferences, workflows across sessions
  • Skills self-improve via GEPA optimizer (ICLR 2026 Oral) using execution traces
  • Single gateway: Telegram, Discord, Slack, WhatsApp, Signal, CLI simultaneously
  • Isolated subagents with their own terminals and Python RPC scripts
  • Model-agnostic — Nous Portal, OpenRouter, OpenAI, local endpoint; switch with one command
  • Can import full OpenClaw settings, memories, and skills via migration wizard
  • 40+ built-in tools — web, browser automation, vision, TTS, cron scheduler, code execution

The differentiator: Hermes Agent is also a research platform — it exports tool-calling trajectories for fine-tuning, integrates with Atropos for RL experiments, and supports 11 tool-call parsers for training any model architecture.

How the Stack Actually Fits Together

These aren't competing replacements for each other. They are complementary layers of a single emerging stack — each solving a different level of the same problem.

1
Natural Language Goal
"Order flowers for my parents" · "Summarise this repo" · "Monitor my server and alert me"
2
Agent Platform — OpenClaw / Hermes
ReAct loop · Tool use · Memory · Subagents · Session transcripts · Skill library
3
Security Layer — NemoClaw (enterprise)
Sandbox isolation · Policy enforcement · Privacy router · Egress control
4
Model Backend
Nemotron (local) · Claude / GPT (cloud via privacy router) · Any open-weight model
🧠

The model is Layer 4 — the foundation, not the ceiling. The agent's observable intelligence is mostly determined by Layers 2 and 3. A mediocre model in a great harness will routinely outperform a great model in a bare chat interface.

Perspective

The Three Tensions Driving Everything

Beneath the product names, three fundamental tensions are shaping how this space evolves. Understanding them is more useful than tracking any single release.

① Capability vs. Safety

OpenClaw proved you can build an extraordinarily capable local agent. The security incident proved that "open and powerful" without sandboxing is dangerous at enterprise scale. NemoClaw is the answer — but the tension between "do more" and "do safely" will never fully resolve. Every new capability creates a new attack surface.

② General vs. Specialised

Coding harnesses (Claude Code, Codex) are optimised for one workload. General platforms (OpenClaw, Hermes) try to be good at everything. History suggests specialised systems win early, then general systems win at scale — once the infrastructure matures. We're somewhere in the middle of that curve right now.

③ Cloud vs. On-Device

All three current platforms assume a persistent server or desktop machine. The next frontier — running always-on, capable agents on mobile and edge devices with NPU acceleration — is still largely unsolved. The constraints are brutal: smaller context windows, tighter memory budgets, intermittent connectivity, battery awareness. But the user value is enormous.

"Mac and Windows are the operating systems for the personal computer. OpenClaw is the operating system for personal AI. This is the moment the industry has been waiting for — the beginning of a new renaissance in software."
— Jensen Huang, CEO NVIDIA · GTC 2026 Keynote

That framing matters. If OpenClaw is the Linux of personal AI — open, foundational, built upon by everyone — then NemoClaw is the enterprise distribution (think Red Hat), and Hermes is the power-user desktop environment (Arch + custom WM). The model underneath is just the kernel.

And if that analogy holds, we're in approximately 1995 of this arc. The primitives exist. The ecosystem is forming. The killer applications — the things that will make agentic AI feel as inevitable as the web browser — are still being discovered.

Bottom Line

The Model Is the Engine.
The Harness Is the Car.

OpenClaw established the platform. NemoClaw made it enterprise-safe. Hermes made it personal and self-improving. The coding harnesses showed what a specialised, well-engineered scaffold can do to any model. The next decade of AI progress will be written at this layer — not in the transformer architecture itself.

Agents > Models Harness = Differentiator Edge is the Frontier
